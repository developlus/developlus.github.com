Web UIで執筆するCMSやブログシステムに耐えられなくて，様々なドキュメント管理ツールを試した結果，ReST（reStructured Text）にたどり着きました．

ReSTを整形するツールである「Sphinx」，さらにSphinxを利用するブログ構築支援ツール「Tinkerer」を使って，今後のドキュメントを管理していくことにしました．

Tinkerer

http://tinkerer.me/

コマンド一つでブログのテンプレート作成，ドキュメントのビルドを実行することが可能で，生成されるドキュメントは性的なものなので，どこにでもバックアップ，引っ越しが可能です．

GitHub

https://github.com/

言わずと知れた分散リポジトリ管理システムです．

無料で個人のリポジトリを作成できることから巷でも人気のようですが，自分で書いたソースコードをインターネット上に晒す勇気がなくて躊躇していました．お仕事で書いたソースコードは，オンプレで構築されたSubversionで管理していましたが，Tinkererの情報を集めていくうちに，Tinkererで静的なドキュメントを書いた場合は，GitHubによるコンテンツ公開に適しているとのこと．

記事を手元で書いて，GitHubにコミット，そしてWebサーバでPullして最新化することも可能，GitHubをマスターとして記事を管理していれば，もはや記事のバックアップすら不要です．

Bitbucket

GitHubと同様のインタフェースを備える分散リポジトリ管理システム．GitHubとの大きな違いは，「ソースコードの公開が必須ではない」に尽きます．

基本的にはGitHubと同じなので，公開したくなソースコードのリポジトリ管理については，Bitbucketが最適です．
